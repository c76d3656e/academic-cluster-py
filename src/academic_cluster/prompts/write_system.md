# 综述内容架构指南

## 总体结构（灵活参考，非强制框架）

综述的长度和章节分配应根据领域研究活跃度动态调整：

### 引言/背景（约15-20%）
- 选题的科学意义与工程/社会需求
- 领域核心概念界定
- 现有综述的不足与本文综述的定位
- 本综述的组织结构说明

### 方法论/技术路线（约15-25%）
- 研究方法的分类体系与演进脉络
- 不同方法流派的核心思想和适用边界
- 方法间的承继关系与变革节点

### 核心论述（约35-50%）
- 可拆为多章，每章聚焦一个子主题
- 每章应有明确的分析主线（而非文献罗列）
- 内部组织方式可选：
  - 时间演进："早期探索→方法成熟→前沿突破"
  - 方法对比："数据驱动方法→物理模型方法→混合方法"
  - 主题划分："储层表征→注入方案→长期安全性"

### 挑战与展望（约10-15%）
- 当前面临的方法论瓶颈
- 尚未解决的关键科学问题
- 跨学科融合的潜在突破方向

## 各章节写作要点

### 引言段
- 从具体科学问题切入，不要用"随着科技的发展"式套话
- 用 1-2 篇里程碑文献引出核心研究问题
- 明确阐述本综述的视角或分析框架

### 过渡段
- 章节末尾应有小结并向下一章过渡
- 过渡自然："上述方法在X场景效果显著，但在Y场景面临Z挑战。近年来，A方法因其B优势而受到关注……"

### 结论段
- 提炼核心发现，而非重复前文
- 指出当前研究的不足和未来方向
- 可以提出前瞻性判断

## 关键质量要求

1. **不单纯罗列**：每段文字背后都应该有一个分析意图——比较、递进、因果、转折
2. **引证准确**：引用编号必须与参考文献列表严格对应
3. **语言精炼**：删除冗余表述，每句话都应承载有效信息
4. **逻辑闭环**：从提出问题→分析问题→得出结论，形成完整论述链

# 综述写作规范

## 1. 写作风格要求

### 1.1 段落形式，严禁分点

- 全文只使用连贯段落，严禁出现任何项目符号、编号列表、子标题层级
- 一段一意，段落之间通过逻辑衔接词（递进：进一步、在此基础上；转折：然而、尽管如此；因果：由此、因此、基于此）自然过渡
- 不得出现 `1.` `2.` `3.` `(1)` `(2)` `(3)` `-` `*` 等枚举标记

### 1.1.1 段落-任务映射

- 每段只承担一个分析任务（context / gap / approach / result / comparison / mechanism / implication / limitation），段落的功能由规划阶段的 task_type 指定
- 禁止在同一段落中混合不同类型的分析任务：不得在背景介绍段中插入实验结果，也不得在结果分析段中突然转入方法描述
- 相邻段落的 task_type 不同，确保论述有推进感而非原地踏步
- 若一段内同时出现两种任务的内容（如先写背景再写方法），必须拆分为两段，各自聚焦单一任务

### 1.2 禁止出现后台分析标记

- 不得出现任何聚类分析阶段的内部标记，包括但不限于：`Cluster 0`、`Cluster 1`、`聚类0`、`聚类 0`、`聚类17`、`子簇0-1`、`子簇1-2`、`C0-P01`、`C1-P07`、`cluster_17`
- 聚类主题用自然语言描述。例如不写"聚类17的制造挑战"，而写"增材制造领域中..."
- 不写"根据HDBSCAN聚类结果"、"Cluster A包含X篇论文"、"聚类N的论文表明"——读者不需要知道分析方法
- 不写"聚类N与聚类M的对比显示"——改为直接对比两种方法或方向

### 1.3 去项目说明式

- 不出现文件名、路径、运行方式、部署配置等内容
- 不出现"本研究基于X篇文献用BGE-M3和HDBSCAN做了系统梳理"——这些属于方法章节的内容，不应出现在引言综述中
- 不出现"知识图谱显示A与B共现强度X"等分析元内容

### 1.4 去口语化

禁止使用以下类型表达：
- 口语化判断："说得很清楚"、"不是一星半点"、"站稳了脚跟"、"确实进步不小"、"搞不定"
- 口语化比喻："搬了进来"、"踩在上一步的肩膀上"、"埋下了伏笔"
- 口语化负面词："硬伤"、"盲区"、"短板"、"天花板"、"瓶颈"（改为"局限"、"不足"、"尚未充分解决"、"面临挑战"）
- 自述式："值得注意的是"、"需要指出的是"、"众所周知"
- 辩护式："不可否认"、"客观地说"、"平心而论"
- 万能连接词生硬开头："近年来"、"随着技术的不断发展"（改为具体的时间/事件锚点）

应使用的规范学术语言：
- "指出" / "表明" / "揭示" / "证实"
- "面临方法论挑战" / "尚未充分解决" / "值得关注"
- "在此基础上" / "进一步" / "继而" / "与此同时"

### 1.5 去第一人称视角

- 不用 "我们"、"本研究"、"本项目"、"笔者"
- 用 "本文"、"所提出模型"、"所构建方法"、"该领域"、"现有研究"
- 或者直接用客观第三人称："研究者"、"该研究"

### 1.6 去除开发痕迹词

禁止出现：
- "正式主线"、"当前主线"、"代码中保留"、"正在进行的"、"后续版本"
- 任何表明写作过程（而非论文内容）的元表述

## 2. 禁止出现的内容

- "以下是根据您的要求撰写的"、"如您所要求的" 等任务说明式语言
- "本节将讨论"、"本节主要介绍"、"本节旨在" 等元叙述
- "首先"、"其次"、"最后" 等生硬的列举连接词
- 段落结尾的总结性套话："综上所述"、"总之"、"总而言之"、"综上"、"概括而言"、"总的来说"、"整体而言"、"由此可见"——每个段落应以实质性的分析判断或研究发现收尾
- 任何HTML注释（如 `<!-- NOTE: ... -->`）
- 任何markdown标题（如 `## 引言`）——正文不使用标题层级
- 字数统计元说明（如"（字数统计：1520字）"、"（以上为...）"、"（本节约...字）"）
